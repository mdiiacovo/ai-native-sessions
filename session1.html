Know your tools

The Copilot surface, by job

  • Completions — flow state, boilerplate, "finish my thought"
  • Chat — ask, explain, plan, refactor in place
  • Edits / agent mode — multi-file changes from one instruction
  • Coding agent — hand off a whole issue, get a pull request back
Not just for engineers

Copilot for planning

Turn an epic into structured, buildable work.

  • Break an epic into issues with auto-drafted acceptance criteria
  • A concrete takeaway for PMs & designers, not only developers
  • Clear issues are half the build — and exactly what the coding agent needs
The Acquire edge

Comprehend inherited systems

  • Point Copilot at unfamiliar or acquired code → get a plain-English explanation
  • Ask it to document the module so the team can own it
  • Turns "who wrote this?" into "here's what it does" in minutes
  • Demo: the undocumented legacy/triage-rules.js in the intake app
The pro move

Grounding with AGENTS.md

Great context in → great output out.

  • Commit an AGENTS.md with 3–4 project rules (stack, conventions, don'ts)
  • Run the same prompt with it present vs. absent — watch quality jump
  • The thing beginners skip and strong builders never do
Right tool, right job

Which model, when

  • Fast model — quick edits, boilerplate, high-volume iteration
  • Reasoning model — tricky logic, planning, gnarly bugs
Same task, two models — compare the result and the speed.

How the best builders prompt

Best-in-class prompting

  • Specify the outcome, not the steps
  • Give constraints — stack, style, don't-touch
  • Iterate in small loops, not one mega-prompt
  • Ask it to plan first, then build
