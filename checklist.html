Before the Hackathon

A 2-minute readiness check so you can build from minute one · tap items to tick them
💡 This event is browser-first — you can do the whole build on github.com with the Copilot coding agent, no local setup required. The IDE items are optional for those who prefer to code locally.

🔑 Access & entitlement

Copilot seat assignedConfirm you have an active GitHub Copilot license.
Copilot enabled for your accountCheck github.com → Settings → Copilot shows it as active.
Coding agent enabled by org policyYour org must allow the Copilot coding agent so you can assign issues to it.
Repo accessYou can create a repo (or have write access to your team's) to open issues and PRs.

🌐 Browser-first path (recommended)

You can open an issue and assign CopilotIssue → Assignees → Copilot. If Copilot isn't listed, revisit the org policy item above.
Copilot Chat works on github.comTry asking it to explain a file in any repo.
(Optional) GitHub Pages readyIf you want a live URL, enable Pages so merges auto-deploy.

💻 Local / IDE path (optional)

IDE extension installedGitHub Copilot + Copilot Chat in VS Code (or your editor).
Signed in & agent mode availableConfirm agent/edits mode appears in the Copilot panel.
Git configuredYou can clone, commit, and push.
